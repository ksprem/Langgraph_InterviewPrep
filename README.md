# Interview Prep Suggester - LangGraph Learning Project

A LangGraph-based interview preparation assistant that suggests a personalized study plan based on the user's interview context.

The project demonstrates a LangGraph flow that:

```text
[User Input]
      |
      v
understand_interview_context
      |
      +--> suggest_technical -------+
      +--> suggest_behavioural ----+--> pick_best_prep
      +--> confidence_suggestion --+          |
                                         conditional
                                      /              \
                             quick_prep         deep_prep
                                      |              |
                                     END            END
```

---

## What This Project Does

A user enters an interview description such as:

- `I have a software engineer interview tomorrow`
- `I'm preparing for a product manager role and need confidence`
- `I need to get ready for a technical screening soon`

The graph then:

1. Extracts interview context from the user input.
2. Runs three specialist suggestion nodes in parallel:
   - technical preparation
   - behavioral story prep
   - confidence-building habits
3. Fuses all suggestions into a decision node.
4. Selects either a quick practice plan or a deeper preparation session.
5. Prints the final personalized prep plan and message log.

---

## LangGraph Concepts Covered

| Concept | Where It Appears |
|---|---|
| State | `InterviewPrepState` Pydantic model |
| Nodes | `understand_interview_context`, `suggest_technical`, `suggest_behavioural`, `confidence_suggestion`, `pick_best_prep`, `quick_prep`, `deep_prep` |
| Parallel execution | Three suggestion nodes run after `understand_interview_context` |
| Fan-in | `technical_suggestion`, `behavioural_suggestion`, and `confidence_suggestion` all flow into `pick_best_prep` |
| Conditional edges | `route_after_decision` chooses `quick_prep` or `deep_prep` at runtime |
| Final output | `final_plan` from `quick_prep` or `deep_prep` |
| Message accumulation | `messages: Annotated[list, operator.add]` lets node messages merge |

## Setup

### 1. Create and activate a virtual environment

```powershell
python -m venv .venv
.venv\Scripts\activate
```

On macOS/Linux:

```bash
python -m venv .venv
source .venv/bin/activate
```

### 2. Install dependencies

```powershell
pip install -r requirements.txt
```

### 3. Configure your OpenAI API key

```powershell
copy .env.example .env
```

Edit `.env` and add your API key:

```text
OPENAI_API_KEY=sk-...
```

Never commit your real `.env` file.

### 4. Run the project

```powershell
python interview_prep_graph.py
```

## Key Takeaways

1. Graph state carries the data that travels through the app.
2. Nodes are Python functions that read state and return updates.
3. Parallel execution allows multiple experts to run simultaneously.
4. Fan-in combines parallel outputs into a single decision point.
5. Conditional edges route execution based on state.
